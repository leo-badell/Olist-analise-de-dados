# Reflexão Teórica — Limpeza de Dados e Machine Learning

## Como a qualidade dos dados impacta modelos de IA

A limpeza e a padronização dos dados são etapas fundamentais antes de qualquer processo de treinamento em Machine Learning. Quando um modelo é alimentado com dados inconsistentes — como categorias de produto mal preenchidas, campos de dimensão nulos ou strings com formatação irregular — ele aprende padrões que não refletem a realidade, mas sim os erros e ruídos do conjunto de entrada. Isso pode gerar **viés** nas predições: o modelo passa a associar determinadas saídas a artefatos do pré-processamento deficiente em vez de associá-las a características genuínas dos dados. No caso do dataset Olist, por exemplo, produtos sem categoria definida ou com dimensões ausentes poderiam fazer com que um modelo de recomendação ou de previsão de frete ignorasse segmentos inteiros do catálogo, distorcendo suas previsões para esses casos.

A lógica de programação aplicada ao tratamento de dados — como a substituição de valores nulos por defaults semânticos, a normalização de strings via expressões regulares e a aplicação de regras de negócio consistentes — contribui diretamente para reduzir o **overfitting**. Um modelo sobreajustado memoriza as particularidades do conjunto de treinamento, incluindo seus erros; ao limpar e padronizar os dados antes do treino, removemos variações espúrias e tornamos o conjunto mais representativo da distribuição real do problema. Isso faz com que o modelo generalize melhor para dados novos, que é exatamente o que se busca em um sistema de IA confiável e robusto em produção.
